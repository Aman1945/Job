# NON-CONFORMITY PROCEDURE
**NexusOMS Enterprise - ISO 9001:2015 Clause 10.2**  
**Version**: 1.0  
**Effective Date**: February 12, 2026

---

## 1. PURPOSE

Define the process for identifying, documenting, and addressing non-conformities to ensure continuous improvement and compliance with ISO 9001:2015.

---

## 2. SCOPE

Applies to all non-conformities related to:
- Product/service quality
- Process deviations
- Customer complaints
- Audit findings
- Regulatory non-compliance

---

## 3. DEFINITIONS

- **Non-Conformity (NC)**: Failure to meet a requirement
- **Major NC**: Significant deviation affecting product/service quality
- **Minor NC**: Isolated deviation with minimal impact
- **Corrective Action**: Action to eliminate the cause of non-conformity
- **Preventive Action**: Action to eliminate potential causes

---

## 4. NON-CONFORMITY IDENTIFICATION

### 4.1 Sources
- Internal audits
- Customer complaints
- Process monitoring
- Management reviews
- Employee reports
- External audits

### 4.2 Reporting
All employees can report non-conformities via:
- Email: quality@nexusoms.com
- Quality Management System (QMS)
- Direct report to supervisor

---

## 5. NON-CONFORMITY CLASSIFICATION

| Severity | Description | Response Time | Example |
|----------|-------------|---------------|---------|
| **Critical** | Immediate risk to safety/compliance | 24 hours | Data breach, system failure |
| **Major** | Significant impact on quality | 72 hours | Incorrect order processing |
| **Minor** | Limited impact, isolated incident | 7 days | Documentation error |

---

## 6. NON-CONFORMITY HANDLING PROCESS

### Step 1: Identification and Documentation (Day 1)
```
NC ID: NC-YYYY-MM-DD-XXX
Reported By: _______________
Date: _______________
Category: Product / Process / System / Compliance
Severity: Critical / Major / Minor
Description: _______________
Affected Area: _______________
```

### Step 2: Immediate Containment (Day 1-2)
- Isolate affected products/services
- Prevent further non-conforming output
- Notify affected stakeholders
- Document containment actions

### Step 3: Root Cause Analysis (Day 2-5)
**5 Whys Method**:
1. Why did the non-conformity occur? _______________
2. Why? _______________
3. Why? _______________
4. Why? _______________
5. Why? (Root Cause) _______________

**Fishbone Diagram** (if complex):
- Man: _______________
- Machine: _______________
- Method: _______________
- Material: _______________
- Measurement: _______________
- Environment: _______________

### Step 4: Corrective Action Plan (Day 5-7)
```
Corrective Actions:
1. _______________
   Responsible: _______________
   Deadline: _______________
   
2. _______________
   Responsible: _______________
   Deadline: _______________

Preventive Actions:
1. _______________
2. _______________

Resources Required: _______________
Approval: _______________
```

### Step 5: Implementation (Day 7-30)
- Execute corrective actions
- Update procedures/work instructions
- Provide training (if needed)
- Monitor implementation

### Step 6: Verification (Day 30-45)
- Verify effectiveness of actions
- Check for recurrence
- Review metrics/KPIs
- Document results

### Step 7: Closure (Day 45)
```
Verification Results: _______________
Effectiveness: Effective / Partially Effective / Not Effective
Closure Date: _______________
Approved By: _______________
Lessons Learned: _______________
```

---

## 7. ROLES AND RESPONSIBILITIES

| Role | Responsibility |
|------|----------------|
| **Quality Manager** | Overall NC process management, approval of major NCs |
| **Process Owner** | Root cause analysis, corrective action implementation |
| **Department Head** | Resource allocation, action approval |
| **All Employees** | Report NCs, participate in investigations |

---

## 8. NON-CONFORMITY REGISTER

Maintain a register of all NCs with:
- NC ID
- Date reported
- Category and severity
- Description
- Root cause
- Corrective actions
- Status (Open/In Progress/Closed)
- Closure date

**Review Frequency**: Monthly

---

## 9. METRICS AND REPORTING

### 9.1 Key Metrics
- Number of NCs per month
- NC by category (Product/Process/System)
- NC by severity (Critical/Major/Minor)
- Average time to closure
- Recurrence rate
- Effectiveness of corrective actions

### 9.2 Reporting
- **Monthly**: NC summary to management
- **Quarterly**: Trend analysis and improvement opportunities
- **Annually**: NC performance review

---

## 10. CONTINUOUS IMPROVEMENT

### 10.1 Trend Analysis
- Identify recurring issues
- Analyze root causes across NCs
- Implement systemic improvements

### 10.2 Process Improvement
- Update procedures based on lessons learned
- Enhance training programs
- Improve monitoring and controls
